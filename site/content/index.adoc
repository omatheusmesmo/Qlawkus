= Qlawkus

Qlawkus is a set of Quarkus extensions that give you the foundations to build your own personal AI agent in Java. It is the kind of autonomous, always-on assistant that projects like https://github.com/openclaw/openclaw[OpenClaw] and https://github.com/NousResearch/hermes-agent[Hermes] pioneered, rebuilt on the Quarkus stack and strongly leaning on https://docs.quarkiverse.io/quarkus-langchain4j/dev/[Quarkus LangChain4j].

It ships the building blocks of an agent: a dynamic cognition layer (a "soul"), a layered long-term memory, multi-provider messaging (Discord, Telegram) with voice, Google Workspace tools, and a sandboxed shell.

== Nothing is mandatory

Qlawkus is modular. Every capability is a separate Quarkus extension, so you assemble an agent by depending on just the pieces you want and leaving the rest out. Want a Discord bot with long-term memory and a shell, but no Google integration? Add those modules, skip the others. The only hard dependency is the core (`qlawkus-client`) and, through it, Quarkus LangChain4j.

== Two ways to use it

Ready distribution::
The `app` module wires the core, the tools, and the Discord/Telegram adapters into a deployable agent you can run as-is. Start here if you just want a working agent: xref:getting-started.adoc[Getting Started].

Build your own::
Depend on `qlawkus-client` plus whichever tool and messaging extensions you need, and assemble a focused agent of your own. The xref:architecture.adoc[Architecture] page shows how the modules fit together, and the xref:config-reference.adoc[Config Reference] documents every module's settings.

== What's inside

* *Cognition* - a persistent `Soul` (identity, mood, focus) that shapes the agent's behavior.
* *Memory* - layered long-term memory injected into the prompt every turn (owner profile, active retrieval, transcripts, episodic journals), so recall does not depend on the model deciding to search.
* *Messaging* - Discord and Telegram adapters with voice in and out (Slack and WhatsApp adapters exist too).
* *Tools* - Google Workspace (Gmail, Calendar, Drive, Sheets, Storage), a brag-document tool, and a sandboxed shell.
* *Resilience* - a primary model with an Ollama fallback behind a circuit breaker.

== Quick start

[source,bash]
----
# Dev mode (auto-provisions Postgres + Ollama via Dev Services, no .env needed)
mvn install -pl client -am -DskipTests
cd app && mvn quarkus:dev          # http://localhost:8080

# Talk to it
curl -X POST http://localhost:8080/api/chat/sync -u qlawkus:qlawkus \
  -H 'Content-Type: application/json' -d '{"message":"Hello, who are you?"}'
----
