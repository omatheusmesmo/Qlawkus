= Configuration Reference

All runtime configuration is environment-variable driven (mapped from `app/src/main/resources/application.properties`). Defaults shown are the values baked into the image; override any of them in your `.env` / shell, or in the `environment` block of `docker-compose.yml` / `docker-compose.local.yml`.

== LLM models

The agent uses a primary OpenAI-compatible model with an Ollama fallback (circuit breaker on repeated failures). Pick a provider profile with `LLM_KIND`; it sets the base URL plus opinionated chat/embedding defaults for that provider. The Ollama fallback is configured separately.

[cols="2,2,3",options="header"]
|===
|Variable |Default |Description

|`LLM_KIND` |`openai` |Provider profile (see table below). Sets base URL + chat/embedding models + dimensions.
|`LLM_API_KEY` |`dummy` |API key for the selected provider (required in prod).
|`OLLAMA_FALLBACK_MODEL` |`qwen3.5:4b` |Fallback chat model (Ollama).
|`OLLAMA_FALLBACK_EMBEDDING_MODEL` |`mxbai-embed-large` |Fallback embedding model (Ollama).
|===

Built-in provider profiles (all OpenAI-compatible, one `quarkus-langchain4j-openai` dependency):

[cols="1,2,2",options="header"]
|===
|`LLM_KIND` |Chat default |Embedding default

|`openai` |`gpt-5-mini` |`text-embedding-3-large` @1024
|`nvidia` |`nvidia/nemotron-3-ultra-550b-a55b` |`nvidia/nv-embedqa-e5-v5` @1024 (input_type)
|`deepseek` |`deepseek-v4-flash` |`deepseek-embed` @1024
|`mistral` |`mistral-large-latest` |`mistral-embed` @1024
|`groq` |`qwen/qwen3.6-27b` |— (chat only)
|`xai` |`grok-4.3` |— (chat only)
|`openrouter` |`openrouter/owl-alpha` |— (chat only)
|===

=== Overriding a profile

Every value defaults from the kind and stays overridable:

* *Friendly env aliases*: `LLM_BASE_URL`, `LLM_CHAT_MODEL`, `LLM_EMBEDDING_MODEL`, `EMBEDDING_DIMENSION`.
* *Native quarkus-langchain4j keys* also win (e.g. `quarkus.langchain4j.openai."primary".chat-model.model-name`) — Qlawkus does not duplicate them.
* *Different provider for embeddings* (required for chat-only kinds like Groq/xAI/OpenRouter): set `LLM_EMBEDDING_KIND`, e.g. chat on Groq, embeddings on OpenAI.

The embedding model must output the dimension of the pgvector column (default `1024`). A different dimension requires changing `EMBEDDING_DIMENSION` and migrating the `embeddings` table.

== Databases

Three PostgreSQL databases (all Flyway-managed): `qlawkus` (main: soul, chat history, journals, embeddings), `qlawkus_google_auth`, and `qlawkus_brag`.

[cols="2,2,3",options="header"]
|===
|Variable |Default |Description

|`POSTGRES_DB` |`qlawkus` |Main database name.
|`POSTGRES_USER` |`qlawkus` |Database user (shared across the three DBs).
|`POSTGRES_PASSWORD` |`qlawkus` |Database password.
|===

In Docker, `docker/postgres-init/01-create-databases.sql` creates the auxiliary databases.

== Agent memory

The memory subsystem (see xref:architecture.adoc[Architecture]). Recall is injected into the prompt, so these mostly tune retrieval breadth and the background jobs.

[cols="2,1,3",options="header"]
|===
|Variable |Default |Description

|`AGENT_SHARED_CONTEXT` |`true` |One shared conversation across all channels (the owner is a single user).
|`AGENT_CONTEXT_TTL_MINUTES` |`60` |Shared-context idle TTL.
|`AGENT_MEMORY_MAX_MESSAGES` |`40` |Working-memory window size (messages kept verbatim).
|`AGENT_MEMORY_MIN_SCORE` |`0.7` |Min cosine score for the `searchMemories` tool.
|`AGENT_MEMORY_MAX_RESULTS` |`10` |Max results for `searchMemories`.
|`AGENT_ACTIVE_MEMORY` |`true` |Inject query-relevant facts before every reply (no tool call needed).
|`AGENT_ACTIVE_MEMORY_MAX_RESULTS` |`5` |Facts injected per turn.
|`AGENT_ACTIVE_MEMORY_MIN_SCORE` |`0.7` |Min cosine score for active-memory retrieval.
|`AGENT_TRANSCRIPT_ARCHIVE` |`true` |Archive each message as a searchable transcript embedding.
|`AGENT_TRANSCRIPT_MAX_RESULTS` |`5` |Max results for `searchTranscripts`.
|`AGENT_TRANSCRIPT_MIN_SCORE` |`0.7` |Min cosine score for transcript search.
|`AGENT_MEMORY_REVIEW_SIMILARITY` |`0.97` |Cosine similarity above which the nightly job treats two facts as duplicates.
|`AGENT_MEMORY_CURATION` |`true` |Nightly job that folds facts into the owner profile (uses the LLM).
|===

NOTE: The embedding model is retrieval-tuned, so relevant pairs score high; `0.7` is a deliberate safety margin, not a strict cutoff.

== Messaging

[cols="2,1,3",options="header"]
|===
|Variable |Default |Description

|`DISCORD_BOT_TOKEN` |(empty) |Enables the Discord bot when set.
|`DISCORD_APP_ID` |(empty) |Discord application id (for slash commands).
|`DISCORD_GUILD_ID` |(empty) |Optional: scope slash commands to one guild.
|`DISCORD_STARTUP_CHANNEL_ID` |(empty) |Optional: post a startup message here.
|`DISCORD_ALLOWED_USER_IDS` |`*` |Comma-separated allowlist; `*` or empty = everyone.
|`DISCORD_RESPOND_ALL` |`true` |If false, only respond to DMs and slash commands.
|`TELEGRAM_BOT_TOKEN` |(empty) |Enables the Telegram bot when set.
|`TELEGRAM_MODE` |`polling` |`polling` or `webhook`.
|`TELEGRAM_ALLOWED_USER_IDS` |`*` |Allowlist; `*` or empty = everyone.
|===

IMPORTANT: The Discord adapter requires the *Message Content Intent* enabled in the developer portal, otherwise it connects but cannot read messages.

== Voice (optional)

|`WHISPER_API_KEY` / `GROQ_API_KEY` |(empty) |Speech-to-text key (transcription of voice notes).
