= Quick Start
:page-title: Quick Start
:page-description: Get Qlawkus running in minutes

== Prerequisites

* Docker -- the only requirement for the containerized run modes below; their images compile the whole project in-container
* JDK 25 and Maven -- only for dev mode (`quarkus:dev`) and building from source directly
* An `LLM_API_KEY` for your chosen provider (e.g. NVIDIA from https://build.nvidia.com/) -- only needed for the production profile, not for dev mode

== Clone and run

[source,bash]
----
git clone https://github.com/omatheusmesmo/qlawkus.git
cd qlawkus
mvn install -pl client -am -DskipTests
cd app && mvn quarkus:dev
----

Dev Services auto-provision PostgreSQL (pgvector) and Ollama via Testcontainers. No `.env` required.

The agent starts on `http://localhost:8080`. In dev mode, Flyway resets all databases on every restart.

TIP: The first build takes a few minutes. Subsequent starts are much faster.

== Try it out

[source,bash]
----
curl -X POST http://localhost:8080/api/chat/sync -u qlawkus:qlawkus \
  -H 'Content-Type: application/json' -d '{"message":"Hello, who are you?"}'
----

== Other run modes

The containerized modes build the whole reactor *inside* Docker (a multi-stage
build on the Red Hat UBI OpenJDK image), so they need only Docker -- no host JDK
or Maven. The `run.sh` helper wraps Docker Compose; run `./run.sh --help` for the
full list of modes and actions.

NOTE: The first in-container build downloads the full dependency tree, so it takes
a few minutes. A cached Maven repo (a BuildKit cache mount) makes later builds fast.

=== Local Docker (Ollama + PostgreSQL)

Self-contained, no external API key required.

[source,bash]
----
./run.sh local
----

Exposes `http://localhost:8742`. An `ollama-init` one-shot container pulls the models on first start.

=== Production Docker (bring your own LLM + PostgreSQL)

Set `LLM_API_KEY` in `.env`. The Compose files ship `LLM_KIND=nvidia`; override `LLM_KIND` to pick another provider (the app's own default kind is `openai`).

[source,bash]
----
echo "LLM_KIND=nvidia"      >> .env   # or openai, deepseek, mistral, groq, xai, openrouter
echo "LLM_API_KEY=nvapi-xxxx" >> .env
./run.sh prod
----

The container maps host port *8742* to container `8080` (the Google OAuth redirect URI expects 8742).

For the native image (faster startup, lower memory; the native compile is heavy -- minutes and several GB of RAM):

[source,bash]
----
./run.sh native
----

TIP: `./run.sh prod logs` follows the app logs and `./run.sh prod down` stops the stack.

== Authentication

All HTTP endpoints use Basic auth (`quawkus` / `qlawkus` by default, override with `API_USER_PASSWORD`).
